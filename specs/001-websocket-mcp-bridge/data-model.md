# Data Model: MCP Bridge Service

## Entities

### Event
Represents an atomic occurrence from an agent or system component.

| Field | Type | Description |
|-------|------|-------------|
| `id` | UUID | Unique identifier (generated by source) |
| `source` | Enum | One of `ralph`, `multiclaude`, `system` |
| `type` | String | Event category (e.g., `agent_started`, `error`) |
| `ts` | RFC3339 | Timestamp when the event occurred |
| `payload` | Object | Flexible JSON data payload |
| `acked` | Boolean | Whether the event has been acknowledged |
| `acked_by` | String | Identity of the acknowledging agent (optional) |
| `acked_ts` | RFC3339 | Timestamp of acknowledgment (optional) |

### RingBuffer
Internal storage structure for maintaining the last 2000 events.

- **Storage Strategy**: In-memory slice with head/tail pointers or a fixed-size queue.
- **Eviction Policy**: First-In-First-Out (FIFO) when capacity is reached.

## Relationships
- **Source → Event**: One source can produce many events.
- **Event → Ack**: One event can have at most one acknowledgment state transition.

## Validation Rules
- `id`, `source`, `type`, `ts` MUST NOT be empty.
- `ts` MUST be a valid RFC3339 timestamp.
- `source` MUST be one of the pre-defined enums.
- `payload` MUST be a valid JSON object.
